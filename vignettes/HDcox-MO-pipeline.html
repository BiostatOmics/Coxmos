Introduction

The HDcox R package contains the necessary functions to reproduce the pipeline in this paper*, a study by Salguero-Garcia et al. in which we analyze ‚Ä¶.

The pipeline includes ‚Ä¶ basic analysis blocks:

  1. Cross-validation and performing survival models for high dimensional data sets. First, ‚Ä¶

  2. Comparing multiple survival models. ‚Ä¶

  3. Understanding the results in terms of the original variables. ‚Ä¶

Installation

HDcox can be installed from GitHub using devtools:

install.packages("devtools")
devtools::install_github("ConesaLab/HDcox", build_vignettes = TRUE)

Getting ready

To run the analyses in this vignette, you‚Äôll first need to load HDcox:

# load HDcox
library(HDcox)

In addition, we‚Äôll require some additional packages for data formatting. Most of them are signaled as HDcox dependencies, so they will already be installed in your system.

To generate plots, we make use of the RColorConesa R package. After install:

# install.packages("devtools")
devtools::install_github("ConesaLab/RColorConesa")

‚Ä¶you can load and set the theme of your R session as follows:

library(RColorConesa)

Input data

The HDcox pipeline requires two matrices as input. First one must be the features under study, and the second one a matrix with two columns called time and event for survival information.

To generate the matrices, we first processed the data from ‚Ä¶ Details to this process can be found in our manuscript (see Supplementary Note and Methods).

If you wish to reproduce the analyses in Salguero-Garcia et al.¬†[1]), you can load the mo.data_Glioblastoma object to use our dataset:

Cross-validation and performing survival models for high dimensional data sets

‚Ä¶

HDcox provides multiple functions with different methodologies.

  • Classical approaches: cox, coxSW and coxEN
  • PLS approaces: plsicox, splsdrcox, splsdrcox_mixOmics and plsdacox_mixOmics

More information for each approach ‚Ä¶

And also, we are going to split the data into train/test (70-30 %).

TRAIN - TEST

set.seed(123)
index_train <- caret::createDataPartition(Y$event,
                                          p = .7, #70% train
                                          list = FALSE,
                                          times = 1)

X_train <- list()
X_test <- list()
for(omic in names(X)){
  X_train[[omic]] <- X[[omic]][index_train,,drop=F]
  X_test[[omic]] <- X[[omic]][-index_train,,drop=F]
}

Y_train <- Y[index_train,]
Y_test <- Y[-index_train,]

EVP per block:

for(b in names(X_train)){
  EPV <- sum(Y_train$event==1) / ncol(X_train[[b]])

}

As classical and PLS approach can not be perform in multiomic data, several multiomic methods have been designed. The methods are divided into two categories:

  1. Single-Block approach. ‚Ä¶
  2. MultiOmic-Block approach. ‚Ä¶

But first, we establish the center and scale for each block:

x.center = c(genes = T, miRNA = T, clinical = T) #if vector, must be named
